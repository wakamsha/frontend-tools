# Comparison

ESLint が提供するルールのうち、oxlint がまだ提供できていないルールがどれくらいあるのかを比較・検証するためのワークスペースです。

昨今は ESLint から oxlint への移行を前向きに検討するプロジェクトが増えていますが、移行によってそれまで依存していた Lint ルールが失われてしまうことを懸念されるケースは少なくありません。

oxlint は ESLint との高い互換性を強く意識したツールであり、圧倒的なスピードで提供するルールを増やしていますが、完全に ESLint と同等のルールセットが提供されるには至っていません。

また、ESLint が提供するルールの中には既に非推奨となっていたり、メンテナンスが凍結されているものも存在します。それらは古いコードの文法を対象としたものであったり、TypeScript など他のツールが提供する機能と重複しているため、利用事例が減少傾向にあることが考えられます。

このワークスペースでは、oxlint がまだ提供できていない ESLint のルールをリストアップし、どのようなルールが提供されていないのか、またそれらのルールが私たちのプロジェクトにとってどの程度重要であるのかを検証することを目的としています。仮にリストアップされた未提供ルールのうち**重要でないものが多い場合、**oxlint への移行に対する懸念が軽減されることが期待できます。

## 移行検討レポート

未提供ルールの概要と移行影響度を整理した資料を [`docs/migration-report.md`](./docs/migration-report.md) にまとめています。各ルールが何を検出するのか、移行によって失われるルールがどの程度の懸念になるのかを、カテゴリ別・重要度別に確認できます。
